Special Comments
SC1) The field final indicates whether the transaction can be updated by non admin users.  A true value means it cannot be updated by non admin users.
SC2) These fields are read only because payments inherit these items from their linked pledge.  This means these items will be ignored on any add or update.
SC3) The field letter must match (case sensitive) an existing letter name in eTapestry.  Existing letter names can be found at Manage...Letters.
SC4) The field pledgeRef indicates the pledge transaction this payment belongs to.  When an update is processed, the pledgeRef is ignored because a payment cannot be switched or removed from its existing linked pledge.
SC5) The field softCredit contains the soft credit transaction linked to this payment.  The linked soft credit can have a different fund, campaign, approach, letter, and amount.  Please note, soft credit transactions can only belong to donor role accounts.  If an update is processed, the soft credit account can be switched from one account to another.
SC6) The field tributeAccountRef indicates the transaction is in honor/memorial of a particular account.  This field is read only for payments because they inherit their pledge's tribute.
SC7) The field recognitionName should only be populated when recognitionType is 3.
SC8) eTapestry allows transactions to be reversed.  The field offsettingRef indicates this transaction is a transaction that has been reversed or the reversal transaction.  Reversal transanctions or transactions that have been reveresed cannot be updated.
SC9) eTapestry allows segmented donation transactions which consist of multiple gifts and/or payments.  The field segmentedTransactionRef indicates this payment is a single segment of a segmented donation transaction.  Individual segments of a segmented donation transaction cannot be updated, but the entire segmented donation transaction (which includes the individual segments) can be updated.
SC10) eTapestry allows transactions to denote a tribute (tributes are separate accounts within the database).  For reporting purposes, eTapestry stores a replica of the transaction within the tribute's journal.  When the field originalAccountRef, originalAccountName, and originalTransactionRef are populated, this transaction is a tribute's replica.  The field originalAccountRef will contain the account ref of the original transaction owner.  The field originalAccountName will contain the account name of the original transaction owner.  The field originalTransactionRef will contain the ref of the original transaction.  Please note, replica tribute transactions cannot be updated.  If attempted, soap fault 107 will occur.
SC11) Journal entries retrieved from eTapestry will contain all attachment data except the fileContent.  This is done for efficiency purposes.
