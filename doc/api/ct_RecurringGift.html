Special Comments
SC1) The field final indicates whether the transaction can be updated by non admin users.  A true value means it cannot be updated by non admin users.
SC2) The field fund must match (case sensitive) an existing fund name in eTapestry.  Existing fund names can be found at Manage...Funds.
SC3) The field campaign must match (case sensitive) an existing campaign name in eTapestry.  Existing campaign names can be found at Manage...Campaigns.
SC4) The field approach must match (case sensitive) an existing approach name in eTapestry.  Existing approach names can be found at Manage...Approaches.
SC5) The field letter must match (case sensitive) an existing letter name in eTapestry.  Existing letter names can be found at Manage...Letters.
SC6) The field recurringGiftScheduleRef indicates the recurring gift schedule transaction this recurring gift belongs to.  When an update is processed, the recurringGiftScheduleRef is ignored because a recurring gift cannot be switched or removed from its existing linked recurring gift schedule.
SC7) The field softCredit contains the soft credit transaction linked to this recurring gift.  The linked soft credit can have a different fund, campaign, approach, letter, and amount.  Please note, soft credit transactions can only belong to donor role accounts.  If an update is processed, the soft credit account can be switched from one account to another.
SC8) The field tributeAccountRef indicates the transaction is in honor/memorial of a particular account.  This must be a tribute role account.
SC9) The field recognitionName should only be populated when recognitionType is 3.
SC10) eTapestry allows transactions to be reversed.  The field offsettingRef indicates this transaction is a transaction that has been reversed or the reversal transaction.  Reversal transanctions or transactions that have been reveresed cannot be updated.
SC11) eTapestry allows transactions to denote a tribute (tributes are separate accounts within the database).  For reporting purposes, eTapestry stores a replica of the transaction within the tribute's journal.  When the field originalAccountRef, originalAccountName, and originalTransactionRef are populated, this transaction is a tribute's replica.  The field originalAccountRef will contain the account ref of the original transaction owner.  The field originalAccountName will contain the account name of the original transaction owner.  The field originalTransactionRef will contain the ref of the original transaction.  Please note, replica tribute transactions cannot be updated.  If attempted, soap fault 107 will occur.
SC12) Journal entries retrieved from eTapestry will contain all attachment data except the fileContent.  This is done for efficiency purposes.
