Special Comments
SC1) The field final indicates whether the transaction can be updated by non admin users.  A true value means it cannot be updated by non admin users.
SC2) If an add or update is being performed and this soft credit is linked to a hard credit transaction, the date will be overwritten by the date of the hard credit transaction.
SC3) The field fund must match (case sensitive) an existing fund name in eTapestry.  Existing fund names can be found at Manage...Funds.
SC4) The field campaign must match (case sensitive) an existing campaign name in eTapestry.  Existing campaign names can be found at Manage...Campaigns.
SC5) The field approach must match (case sensitive) an existing approach name in eTapestry.  Existing approach names can be found at Manage...Approaches.
SC6) The field letter must match (case sensitive) an existing letter name in eTapestry.  Existing letter names can be found at Manage...Letters.
SC7) The field hardCreditRef indicates this soft credit is linked to another transaction.  The field hardCreditAccountRef will contain the account ref of the other transaction owner.  The field hardCreditAccountName will contain the account name of the other transaction owner.
