getDuplicateAccount Method
Order Name Type
Return account Account
Input 1 duplicateAccountSearch DuplicateAccountSearch

Comments
1) An Account will be returned based upon the search content provided within DuplicateAccountSearch.  Null will be returned if no account matches the search content provided.
2) Important Note: DuplicateAccountSearch provides a way to locate an existing account within eTapestry using name, address, email address, and phone number. At a minimum, one of the following four critieria must be provided to conduct a basic duplicate account search.  This is done to ensure enough information is provided to make the search tough enough to alleviate inappropriate results.  In essence, locating duplicates simply on name alone isn't appropirate because multiple accounts in the database may have the same name. eTapestry will attempt to locate an account with all of the data provided. If no account contains all of the data provided, eTapestry will attempt to locate an account by each smaller criteria combinations below as appropirate.  This means a search based upon name, address, and email will attempt to locate an account with all three items and if not found, a name/email, a name/address, and email (see DuplicateAccountSearch for more details about matching on email alone) search will also be conducted.
  • Name & Email
  • Name & Address
  • Name & Phone
  • Email
3) Using the DuplicateAccountSearch details provided, if eTapestry locates two or more accounts, a random account will be returned.  See the method getDuplicateAccounts if you prefer to have multiple accounts returned instead of a single account.
4) A soap fault (100 - Missing Duplicate Search Criteria) will occur if one of the four above basic searches isn't found.
