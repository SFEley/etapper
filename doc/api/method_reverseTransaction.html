reverseTransaction Method
Order Name Type
Return transactionRef string
Input 1 transactionRef string
Input 2 note string

Comments
1) This method will reverse a transaction (i.e. create a copy of the transaction with an inverse amount) and add this new transaction to the journal of the original transaction's owner.  The new transaction is called the reversal and the original is considered the reversed.
2) A soap fault will occur if the parameter transactionRef is empty or invalid.
3) The note provided will be added to the note field of the newly created transaction.
4) The following transaction types cannot be reversed: RecurringGiftSchedule and SoftCredit.  Needless to say, reversed or reversal transactions cannot be reversed.
