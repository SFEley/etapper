Special Comments
SC1) The field final indicates whether the transaction can be updated by non admin users.  A true value means it cannot be updated by non admin users.
SC2) The field upgraded indicates this segmented pledge was upgraded (e.g. the pledge amount was increased via increased scheduled payment amounts or additional scheduled payments) at some point.
SC3) The field totalAmount must equal the sum of the amounts on the segments.
SC4) The field nonDeductibleAmount must equal the sum of the non deductible amounts on the segments.
SC5) The field segments must contain two or more pledges.  This field can only contain pledges.
SC6) The field consolidatedCustomSchedule represents a compressed payment schedule that includes all scheduled payments within all pledge segments on this segmented pledge transaction.  If two or more pledge segments have a schedule payment due on the same date, those scheduled payments will be combined into one within the consolidatedCustomSchedule.
SC7) eTapestry allows transactions to be reversed.  The field offsettingRef indicates this transaction is a transaction that has been reversed or the reversal transaction.  Reversal transanctions or transactions that have been reveresed cannot be updated.
