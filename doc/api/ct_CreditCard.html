Special Comments
SC1) The field cardType must match (case sensitive) an existing card type name in eTapestry.  Existing card type names can be found at Manage...System Defined Fields...Credit Card.
SC2) The field cvv2 will not be stored within eTapestry.  This field is only used when addAndProcess is performed.
SC3) These fields are not stored within eTapestry when the corresponding valuable is a scheduled valuable on a pledge.
SC4) These fields are not stored within eTapestry.  They are used when addAndProcess is performed.  If nameOnCard is empty, these fields will be combined (firstName + middleName + lastName) to set the name on the credit card, which is stored within eTapestry.  If neither these fields or nameOnCard is provided when addAndProcess is performed, the name of the account will be used.
