Special Comments
SC1) eTapestry doesn't store individual name parts (title, first, middle, and last) within separate fields.  Instead, a single field contains the entire name of an account which may be an individual, couple, business, etc...  eTapestry has provided the name parts for convenience.  Either an account name or the name parts must be provided on any account add or update.  If no account name is provided, the name parts will be added together (title + first + middle + last) to form the account name.
SC2) If sortName isn't provided on any account retrieval or update, one will be created using title, first, middle, and last.  If all of the individual name part fields are empty, the sort name will equal the account name.
SC3) These fields will always be empty on any account retrieval  See comment number one to learn why.
SC4) If the field personaType is populated, it must match (case sensitive) an existing persona type name in eTapestry.  Existing persona type names can be found at Manage...System Defined Fields...Persona Types.  If this field is empty when an add is performed, the organizaton's default persona type is used.
SC5) These fields are all linked to the personaType.  eTapestry accounts can have multiple persona types.  These fields are subject to change for a single account based upon the personaType provided.
SC6) The field primaryPersona indicates the personaType provided is primary on any account retrieval  It also indicates the personaType should be primary on any account update.
SC7) The country value provided on an account retrieval is a two character code that represents a country name.  All country codes and names can be found within Manage...System Defined Fields...Country Codes.  A valid two character country code must be provided on any account add or update, else a soap fault will occur.
SC8) The field donorRecognitionName should only be populated when donorRecognitionType is 3.
SC9) The field accountRoleType indicates the role type of the account.  All eTapestry accounts will/can have one, two, or all three account role types.  The donor role is the most common one.  The tribute role is a special role that indicates a gift can be made in honor/memorial of this account (see tributeAccountRef on many of the transaction complex types).  A user role is also a special role that has login ability to the eTapestry application.  An account role type can be added (except for a user role) to an existing eTapestry account by setting this field and processing addAccount.  Please note, the ref will always match either donorRoleRef, tributeRoleRef, or userRoleRef.
