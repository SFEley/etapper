Special Comments
SC1) Either field fieldRef or fieldName must be provided.  If both are provided, only fieldRef will be utilized.  If a new field should be created during an add or update, the fieldRef must be empty.  Please note, if an add or update is processed createFieldAndValues set to false, the field must already apply to the account role type or journal entry type it is being being added or updated. Generally speaking, provided the fieldRef if the user defined field already is encouraged because this will increase the speed of the method call.
SC2) Fields fieldName, displayType and dataType cannot be updated on an existing user defined field.  Fields displayType and dataType only apply when an add or update is being performed, createFieldAndValues is set to true, and a user defined field needs to be created.
SC3) If the field display type is single or multi select, the value must match (case sensitive) an existing value within the field provided.  If the field display type is text, any value is acceptable as long as it appropirate for the field's data type.  Data type text will accept any content.  Data type date will only accept a date as a string with the format of m/d/yyyy.  Data type month + day will only accept a date as a string with the format of m/d.  Data type number will accept any valid number as a string (e.g. 3 or 4.5).  Data type currency will accept any valid number as a string as long as it has two or fewer decimal places (e.g. 5 or 123.99).  Please note, a currency symbol can optionally be added to a data type currency value.
